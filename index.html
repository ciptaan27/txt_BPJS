1. Unggah Berkas TXT

Bisa lebih dari satu file (mis. satu file per bulan) — akan digabung otomatis untuk analisis tren.

Klik atau seret file TXT/CSV ke sini Format: teks berpemisah (delimiter) — koma, titik koma, tab, atau pipa "|"
🔒 Berkas Anda tidak diunggah ke server manapun — seluruh pemrosesan berjalan lokal di browser ini.

2. Pemetaan Kolom

Cocokkan kolom pada berkas Anda dengan kolom yang dikenali sistem. Kolom bertanda wajib diisi.

Laporan Pembersihan Data

    Tingkat Tindak Lanjut per Poli / Ruangan

    Hanya menampilkan poli dengan minimal 5 klaim agar tidak bias oleh sampel kecil.

    Analisis Pareto — DPJP dengan Klaim Perlu Tindak Lanjut Terbanyak

    Selisih Tarif RS vs INA-CBG per Kode CBG

    Nilai positif = tarif RS lebih tinggi dari pembayaran INA-CBG (potensi defisit).

    Uji Chi-Square — Asosiasi Status Klaim dengan Variabel Kategorikal

    Menguji apakah status klaim (Disetujui / Perlu Tindak Lanjut) berasosiasi secara statistik dengan Poli, DPJP, atau Kelas Rawat — bukan sekadar kebetulan sampling.

    Pilih variabel untuk menjalankan uji.

    Uji Kruskal-Wallis — Perbedaan Distribusi Numerik Antar Kelompok

    Uji non-parametrik (tidak mengasumsikan distribusi normal — cocok untuk data biaya/LOS klaim yang umumnya menceng) untuk membandingkan median antar kelompok.

    Pilih variabel untuk menjalankan uji.

    Proyeksi Volume & Biaya Klaim

    Proyeksi linear sederhana berbasis tren bulanan yang tersedia. Untuk model deret waktu yang lebih akurat (mis. Prophet/SARIMA dengan efek musiman), disarankan minimal 24 bulan data historis diproses lewat pipeline Python terpisah.

    Ringkasan untuk Direksi / Stakeholder

    Teks dibuat otomatis dari hasil analisis — silakan sunting sebelum dibagikan.
